# pomo_note 요구사항 임시 초안

작성일: 2026-06-23
상태: 임시 초안
목적: 세션이 끊겨도 다음 작업에서 요구사항 정리를 이어갈 수 있도록 현재까지의 결정과 근거를 보존한다.

## 1. 제품 방향

pomo_note는 뽀모도로 타이머 중심의 데스크탑 앱이다.

핵심 목적은 영상 시청을 집중 세션과 연결하고, 휴식 시간에는 영상과 작업 화면에서 사용자를 확실히 분리해 쉬게 만드는 것이다. 메모 기능은 핵심이 아니라 보조 기능이다.

## 2. MVP 핵심 기능

### 2.1 뽀모도로 타이머

- MVP는 프리셋 기반으로 제공한다.
- 사용자 지정 타이머는 후속 기능으로 둔다.
- 기본 프리셋:
  - 25분 집중 / 5분 짧은 휴식 / 4회 집중 후 15분 긴 휴식
  - 50분 집중 / 10분 짧은 휴식 / 4회 집중 후 30분 긴 휴식
- 긴 휴식은 MVP에 포함한다.
- 이유: 앱의 핵심이 "확실히 쉬게 하는 것"이므로 긴 휴식까지 포함해야 제품 방향과 일관된다.

### 2.2 YouTube 미디어 연동

- MVP는 YouTube 중심으로 구현한다.
- YouTube 링크를 앱에 입력하면 앱 안에서 재생한다.
- 집중 시작 또는 집중 재개 시 기본 동작은 자동 재생이다.
- 휴식 시작 시 YouTube는 자동 일시정지되어야 한다.
- 휴식 종료 후 집중이 재개되면 YouTube는 자동 재생되어야 한다.
- 자동 재생이 브라우저 또는 플레이어 정책으로 막히면 사용자가 직접 재생할 수 있는 버튼을 보여준다.
- Netflix는 MVP에서 제외한다.
- Netflix는 추후 후보 기능으로 남기되, DRM, 로그인 세션, WebView 지원 문제 때문에 앱 내 자동 제어를 보장하지 않는다.

근거:
- YouTube IFrame Player API는 JavaScript 기반 재생 제어를 공식 지원한다.
- Netflix는 DRM과 지원 환경 제약 때문에 일반 데스크탑 WebView 안에서 안정적인 자동 재생/정지 제어를 요구사항으로 잡기 어렵다.

참고 자료:
- YouTube IFrame Player API: https://developers.google.com/youtube/iframe_api_reference

### 2.3 플레이어 표시 방식

- 영상 영역은 메인 앱 안에 기본으로 표시한다.
- 별도 미니 플레이어 창으로 분리할 수 있어야 한다.
- 목표 사용감은 PiP와 비슷한 작은 부유 창이다.
- 메인 화면과 미니 플레이어 모두 독립적으로 "항상 위 고정" 토글을 가진다.

### 2.4 휴식 화면

- 휴식 화면은 선택형으로 제공한다.
- 기본값은 강제 휴식형이다.
- 설정에서 건너뛰기 또는 조기 종료 허용 여부를 바꿀 수 있다.
- 휴식 화면 콘텐츠 옵션:
  - 흰 화면
  - 앱 기본 제공 고양이/강아지 이미지
  - 사용자가 추가한 이미지
  - 랜덤
- 기본 휴식 화면은 흰 화면으로 둔다.
- 기본 이미지 세트는 앱에 포함한다.

### 2.5 체크리스트와 짧은 메모

- 메모는 핵심 기능이 아니다.
- 하단에 체크리스트를 둔다.
- 각 체크리스트 항목에는 짧은 메모를 붙일 수 있다.
- 세션 회고, 긴 노트, 지식 관리 기능은 MVP에서 제외한다.

### 2.6 알림과 사운드

- 집중 시작, 휴식 시작, 휴식 종료/재시작 이벤트에 알림을 제공한다.
- 기본 알람 사운드는 학교 종소리 느낌으로 한다.
- 알람 소리는 켜고 끌 수 있어야 한다.
- OS 알림은 선택 기능으로 제공한다.
- 기본은 앱 내부 전환과 알람 소리 중심이다.

### 2.7 로컬 저장

MVP는 로컬 저장만 제공한다.

저장 대상:
- YouTube 링크
- 마지막 선택 프리셋
- 체크리스트 항목
- 체크리스트 항목별 짧은 메모
- 휴식 화면 설정
- 자동 재생 설정
- 알림 및 사운드 설정
- 항상 위 고정 설정

제외:
- 로그인
- 클라우드 동기화
- 백업 파일 내보내기/가져오기

## 3. 레이아웃 방향

기본 화면은 세로 일자형이다.

순서:
1. 영상 영역
2. 타이머 영역
3. 체크리스트/짧은 메모 영역

각 영역은 크기 조절이 가능해야 한다.
영역 숨기기 기능은 MVP에서 제외한다.

## 4. 운영체제 범위

MVP는 Windows 우선이다.

다만 macOS 확장을 염두에 두고 구조를 잡는다.
Windows 전용 구현에 과하게 묶이지 않도록, 창 제어와 알림 기능은 추상화 가능한 방식으로 설계한다.

## 5. 기술 스택 결정

MVP 기술 스택:
- Electron
- React
- TypeScript

선택 이유:
- 사용자가 React에 익숙하다.
- Rust를 모르는 상태에서 Tauri를 선택하면 MVP 속도가 떨어질 가능성이 크다.
- 이 앱의 핵심 기능인 YouTube iframe 제어, 별도 미니 플레이어 창, 항상 위 고정, 로컬 저장은 Electron에서 JavaScript/TypeScript 중심으로 빠르게 검증하기 좋다.

보류 대안:
- Tauri + React

보류 이유:
- Tauri는 앱 크기와 리소스 면에서 유리하다.
- 하지만 Rust와 WebView 제약을 함께 다뤄야 해서 현재 MVP에는 리스크가 크다.

참고 자료:
- Electron 공식 문서: https://www.electronjs.org/docs/latest/
- Tauri 공식 문서: https://tauri.app/start/

## 6. 현재까지의 주요 결정 로그

- 앱 목적은 노트 중심이 아니라 뽀모도로 타이머 중심으로 정했다.
- Netflix보다 YouTube에 집중하기로 했다.
- YouTube는 앱 내부에서 다루고, Netflix는 MVP에서 제외하기로 했다.
- 영상은 메인 앱 안에서도 보고, 별도 미니 플레이어로도 띄울 수 있게 하기로 했다.
- 휴식 시간에는 YouTube를 자동 일시정지하고, 휴식 종료 후 자동 재생하는 방향으로 정했다.
- 휴식 화면은 선택형으로 하되, 기본은 강제 휴식형으로 정했다.
- 휴식 콘텐츠는 흰 화면, 기본 이미지, 사용자 이미지, 랜덤을 지원하기로 했다.
- 체크리스트는 간단하게 유지하고, 각 항목에 짧은 메모만 붙이는 수준으로 정했다.
- 타이머는 사용자 지정 없이 프리셋 기반으로 시작하기로 했다.
- 긴 휴식은 MVP에 포함하기로 했다.
- 저장은 로컬 저장만 하기로 했다.
- Windows MVP로 시작하고 macOS 확장 가능성을 고려하기로 했다.
- 메인 창과 미니 플레이어 창은 각각 항상 위 고정 토글을 갖기로 했다.
- 기술 스택은 Electron + React + TypeScript로 정했다.

## 7. 아직 확정해야 할 질문

다음 세션에서 이어서 확인할 항목:

- 앱이 시스템 트레이에 상주해야 하는지
- 앱 종료 시 타이머가 계속 돌아가야 하는지, 아니면 종료해야 하는지
- 체크리스트 항목의 완료 기록을 날짜별로 보존할지
- YouTube 링크를 하나만 저장할지, 플레이리스트처럼 여러 개 저장할지
- 휴식 화면을 전체 화면으로 강제할지, 현재 앱 창만 덮을지
- 미니 플레이어 크기와 위치를 저장할지
- 알람 사운드 파일을 MVP에 직접 포함할지, 추후 추가할지

## 8. 임시 범위 판단

MVP에 반드시 포함:
- Electron + React + TypeScript 기반 데스크탑 앱
- YouTube 링크 입력 및 앱 내 재생
- 집중/휴식/긴 휴식 타이머
- 휴식 시작 시 YouTube 자동 일시정지
- 휴식 종료 시 YouTube 자동 재생 시도
- 앱 내부 영상 영역과 미니 플레이어 전환
- 메인 창과 미니 플레이어 각각 항상 위 고정
- 전체 화면 휴식 화면
- 흰 화면, 기본 이미지, 사용자 이미지 휴식 콘텐츠
- 체크리스트와 짧은 메모
- 로컬 저장
- 알람 사운드 켜기/끄기

MVP에서 제외:
- Netflix 앱 내 제어
- 클라우드 동기화
- 로그인
- 사용자 지정 타이머
- 긴 세션 회고
- 백업 파일 내보내기/가져오기
- 모바일 앱
